# [sev0] S000: Metric Takedown

This is the first SEV - yay!

Possible root cause:
- https://github.com/kubernetes-sigs/metrics-server/issues/1014

Symptoms:
- K8s API is up and reachable from network, but it's been seen down
- Metrics Server is down. Pod killed, did not come back up
- Lots of log spew looking for 10.85.0.5, but the pod does not exist
- Ingresses like https://sonarr.xnauts.net are not accessible from network
- Hear high CPU going on due to loud continuous fan
- crio.service looks to be up
- k3s.service restarts periodically

Things that happened:
- Upgraded NixOS version the night before, which did not fail
- Had been using bpftracef on the device the night before; left a bpftrace
  program on all night, killproc.bt
- Restarted node morning after manually when it was noticed Metrics Server was
  down, did not fix the issue

Learnings:
- Metrics are completely down; can't use Prometheus/Loki/Grafana to troubleshoot
- Need an external status page

Remediation:
- Cluster was taken down and reprovisioned from scratch (nix/k3s enable/disable)
  along with deletion of pods/containers in CRI-O. Upon bringup, the cluster was
  still being DoSed by Metrics Server!
- Same as above tried again, this time disabling Metrics Server with
  --disable=metrics-server passed to k3s server. The cluster is online now, with
  no loud fan, though workloads are not deployed yet.
